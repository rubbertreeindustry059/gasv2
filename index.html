⚠️ก่อนเริ่ม: ต้องมีบัญชี Google
Firebase ใช้ Google Account ไปที่ console.firebase.google.com และ Login ด้วย Google Account ของคุณก่อน
ส่วนที่ 1 — สร้าง Firebase Project
เปิด Firebase Console
ไปที่ console.firebase.google.com แล้วกด "Create a project" (หรือ "Add project")
📌 ปุ่มสีน้ำเงินกลางหน้าจอ หรือมุมขวาบน
ตั้งชื่อโปรเจกต์
พิมพ์ชื่อ เช่น mypump-station (ใช้ตัวเล็กและ - ได้) → กด Continue
💡 ชื่อนี้จะกลายเป็น URL ของ database ด้วย ไม่ต้องสวยงาม
ปิด Google Analytics (ไม่จำเป็น)
หน้า "Google Analytics for your Firebase project" → toggle ปิด (ไม่จำเป็นสำหรับระบบนี้) → กด Create project
รอ ~30 วินาที จนขึ้น "Your new project is ready" → กด Continue
ส่วนที่ 2 — เปิดใช้งาน Authentication
เข้าเมนู Authentication
ในหน้า Project Overview → เมนูซ้าย → Build → Authentication → กด "Get started"
เปิด Email/Password Provider
Tab "Sign-in method" → คลิก Email/Password → เปิด toggle "Email/Password" (อันบนสุด) → กด Save
✓ ไม่ต้องเปิด Passwordless / Email link
ส่วนที่ 3 — สร้าง Firestore Database
เข้าเมนู Firestore Database
เมนูซ้าย → Build → Firestore Database → กด "Create database"
เลือก Mode และ Location
• Security rules: เลือก "Start in production mode" (เราจะตั้ง rules เองใน Step ถัดไป)
• Location: เลือก asia-southeast1 (Singapore) → ใกล้ไทยที่สุด ความหน่วงต่ำ
→ กด Enable รอ ~1 นาที
⚡ ถ้าไม่มี singapore ให้เลือก asia-east1 (Taiwan)
ตั้ง Security Rules
Tab "Rules" → ลบโค้ดเก่าออกทั้งหมด → วางโค้ดนี้แทน:
rules_version = '2'; service cloud.firestore { match /databases/{database}/documents { // Settings — อ่านได้ทุกคน, เขียนได้เฉพาะ Admin match /settings/{doc} { allow read: if request.auth != null; allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } // Users — อ่านได้เฉพาะเจ้าของ/Admin match /users/{uid} { allow read: if request.auth != null && (request.auth.uid == uid || get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'); allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } // Pending registrations match /pendingUsers/{doc} { allow read, delete: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; allow create: if true; // anyone can register } // Dispensers match /dispensers/{doc} { allow read: if request.auth != null; allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } // Meter entries — staff write, admin full match /meterEntries/{doc} { allow read: if request.auth != null; allow create: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.active == true; allow update, delete: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin'; } // Stock match /stock/{doc} { allow read: if request.auth != null; allow write: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.active == true; } match /stockLog/{doc} { allow read: if request.auth != null; allow create: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.active == true; } match /fuelReceives/{doc} { allow read: if request.auth != null; allow create: if request.auth != null && get(/databases/$(database)/documents/users/$(request.auth.uid)).data.active == true; } } }
→ กด Publish
🔒 Rules เหล่านี้บังคับว่า Staff ดูข้อมูลได้ แต่แก้ไขตั้งค่าไม่ได้
ส่วนที่ 4 — ลงทะเบียน Web App & ได้ Config
เพิ่ม Web App
กลับไปหน้า Project Overview (กด 🏠 ซ้ายบน) → กดไอคอน </> (Web)
ใส่ชื่อ App Nickname เช่น pump-web → ไม่ต้องเลือก Firebase Hosting → กด Register app
คัดลอก firebaseConfig
จะเห็น code snippet ประมาณนี้ — คัดลอกเฉพาะ object ข้างใน const firebaseConfig = { ... }:
const firebaseConfig = { apiKey: "AIzaSyXXXXXXXXXXXXXXXXXXX", authDomain: "mypump-station.firebaseapp.com", projectId: "mypump-station", storageBucket: "mypump-station.appspot.com", messagingSenderId: "123456789012", appId: "1:123456789012:web:abcdefabcdef" };
→ กด Continue to console
📋 เก็บ config นี้ไว้ — จะนำไปแปะในโค้ด HTML
ส่วนที่ 5 — แก้ไขไฟล์ HTML ให้ใช้ Firebase ของคุณ
เปิดไฟล์ HTML และหา firebaseConfig
เปิดไฟล์ fuel_station_v3.html ด้วย Notepad / VS Code → Ctrl+F ค้นหา YOUR_API_KEY
จะเห็น block:
const firebaseConfig = { apiKey: "YOUR_API_KEY", authDomain: "YOUR_PROJECT_ID.firebaseapp.com", ...
→ แทนที่ทั้ง block ด้วย config ที่คัดลอกมาจาก Step ก่อน
ส่วนที่ 6 — สร้าง Admin Account คนแรก
สร้าง Admin ผ่าน Firebase Console โดยตรง
เพราะยังไม่มี user ใดในระบบ ต้องสร้าง Admin ด้วยมือใน Console

1. เมนูซ้าย → Authentication → Tab "Users" → กด "Add user"
2. Email: admin@pump.local | Password: ตั้งรหัสผ่านที่แข็งแรง → Add user
3. คัดลอก User UID ที่ปรากฏในตาราง (ประมาณ 28 ตัวอักษร)

4. เมนูซ้าย → Firestore Database → กด "+ Start collection"
  Collection ID: users → Next
  Document ID: วาง UID ที่คัดลอกมา
  เพิ่ม Fields:
name (string) → "ผู้จัดการ" username (string) → "admin" email (string) → "admin@pump.local" role (string) → "admin" active (boolean) → true
→ กด Save
🔑 ตอนนี้ login ได้แล้ว ด้วย username: admin และรหัสผ่านที่ตั้งไว้
ส่วนที่ 7 — ทดสอบและเปิดใช้งาน
เปิดไฟล์ HTML ในเบราว์เซอร์
ดับเบิ้ลคลิกไฟล์ fuel_station_v3.html → เปิดใน Chrome / Edge
Login ด้วย username admin และรหัสผ่านที่ตั้งไว้
✓ ถ้าเข้าได้ — Firebase ทำงานปกติ!
ตั้งค่าปั๊มและเพิ่มหัวจ่าย
เมนูซ้าย → ตั้งค่าปั๊ม → กรอกชื่อปั๊ม, ราคาขาย, ราคาทุน → บันทึก
เมนูซ้าย → ตั้งค่าหัวจ่าย → เพิ่มหัวจ่ายทีละตัว
จากนั้นพนักงานสามารถ สมัครใช้งาน ผ่านหน้า Login แล้ว Admin อนุมัติได้เลย
🎁 Firebase Free Tier ให้อะไรบ้าง?
• Firestore: อ่าน 50,000 ครั้ง / วัน | เขียน 20,000 ครั้ง / วัน | เก็บข้อมูล 1 GB
• Authentication: ไม่จำกัด (MAU สูงสุด 50,000 คน/เดือน)
• Bandwidth: 10 GB / เดือน
✓ ปั๊มน้ำมัน 1 สาขา ใช้ฟรีได้ตลอด ไม่ถึงขีดจำกัดแน่นอน
